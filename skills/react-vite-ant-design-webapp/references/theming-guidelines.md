# Theming Guidelines

Use these defaults for consistent Ant Design theming:

- Define semantic tokens first (background, text, primary, muted, border).
- Map semantic tokens to light/dark values through `ConfigProvider` theme algorithms.
- Keep spacing scale limited (for example 0, 4, 8, 12, 16, 24, 32).
- Keep radius scale limited (for example 0, 4, 8, 12, 999).
- Reuse theme tokens across components instead of hardcoded values.
